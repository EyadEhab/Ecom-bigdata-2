Step into the role of a Data Engineer at a high-growth e-commerce startup. The company has accumulated a massive archive of raw, unstructured web-server logs detailing user interactions — clicks, views, cart additions, and purchases. Your objective is to build an interconnected, end-to-end data pipeline across three phases: distributed processing with Spark, NoSQL storage for real-time querying, and an enrichment layer that powers a targeted marketing campaign.

⚠️

The raw dataset is significantly larger than available RAM. Standard iterative loops or in-memory libraries like Pandas will crash with an OOM error. You must use Apache Spark and strictly apply the MapReduce paradigm to distribute computation across CPU cores.

Phase Details

Phase 01

Distributed Data Processing — The MapReduce Paradigm

Task 1.1

Market Basket Analysis — Item Co-occurrence

Objective
Determine which items are most frequently purchased together in the same session across the entire dataset.
Requirement
Formulate a MapReduce strategy that groups items from the same session and reduces them to find the global frequency of co-purchased pairs.
Map Step
Emit (session_id, item_id) pairs for every purchase event in the log.
Reduce Step
Group by session, generate all item pairs, count global frequency of each pair.
Task 1.2

User Affinity Aggregation

Objective
Build a behavioral profile for each user based on their historical actions.
Requirement
Apply MapReduce to assign weighted scores to different user events and aggregate per user.
Weights
View = 1  Cart = 3  Purchase = 5
Output
Each user's top-affinity product categories ranked by weighted score sum.
Phase 02

NoSQL Data Modeling & Storage

Tasks 2.1 & 2.2

Schema Design & Data Ingestion

Objective
Design a data model optimized for instant lookups in real-time web recommendation queries, then load Phase 1 output into it.
Constraint
Relational databases are unsuited for the read-heavy, low-latency requirements of the web frontend. A NoSQL approach is mandatory.
Option A
MongoDB
Design a denormalized document schema. Embed top categories and co-occurrence data directly into user documents for single-query retrieval.
Option B
Redis
Define a Key-Value naming convention and leverage appropriate data structures — Strings, Hashes, or Sorted Sets for ranked affinities.
Phase 03

Data Enrichment & The Interconnected Pipeline

Task 3.1

NoSQL-to-Spark Integration — Cart Abandonment Recovery

Goal
The marketing team wants to send high-value discount codes only when the abandoned item belongs to a user's top affinity categories — saving budget by targeting intelligently.
  • 01
    Read a new batch of raw logs using Spark and filter for users who added an item to their cart but did not trigger a purchase event in that session.
  • 02
    Configure Spark to query historical User Profiles directly from your chosen NoSQL database (MongoDB connector or Redis client).
  • 03
    Join the two datasets using Spark transformations. If the abandoned item's category matches one of the user's top categories → flag High_Discount. Otherwise → flag Standard_Reminder.
